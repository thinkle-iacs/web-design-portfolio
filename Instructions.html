I should have given you a template which refers to this sample portfolio .

That page includes code to automatically pull in descriptions of standards and to make it easy for you to include code snippets in your portfolio.

For your portfolio, you will be demonstrating mastery of each skill listed here. The list of skills span both coding concepts and design concepts.

In each case, you will want to include:

  • Links to full projects showing off the skills in question.
  • Code snippets highlighting your understanding of the concepts/skills you are supposed to demonstrate.
  • A brief reflection about the skill (as needed). For simple skills, there's not need to explain the obvious; but for more open-ended items, such as items highlighting design sense, you should explain your thinking.

I recommend that you organize your portfolio into sections, with each <section> including the standards you are addressing and showing work and code excerpts highlighting your work.

To insert my "standard" blurb, you simply need to add the correct code to a site with my demo script on it. For example, 

<div class="descriptor data-ref="code/basic-html"></div>
will insert the blurb about basic HTML.

You should also customize the look and feel of your portfolio as you move through the course, making it grow more personalized and unique over time.
